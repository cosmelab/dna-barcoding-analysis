Understanding Consensus Sequences

Sanger sequencing reads DNA from one direction at a time. By sequencing both forward (F) and reverse (R) strands, we can create more accurate consensus sequences.

What you'll see below:

  • Forward Read: Original 5' â†’ 3' sequence
  • Reverse Read (Original): Raw reverse strand as sequenced
  • Reverse Read (Rev Comp): Reverse read converted to match forward orientation
  • Consensus Sequence: The selected sequence used for analysis (longer/better quality)

Consensus Strategy:

  • Forward and reverse reads are compared
  • The longer sequence is selected as consensus
  • This provides better coverage of the target region
  • All sequences below are shown in full length for your review

Understanding Consensus Sequences

Why create consensus sequences?

Sanger sequencing reads DNA from one direction at a time. By sequencing both forward (5' to 3') and reverse (3' to 5') strands, we can:

  • Improve accuracy: Combine information from both directions
  • Extend coverage: Forward and reverse reads may cover different regions
  • Validate base calls: Confirming bases from both strands reduces errors

How is the consensus created?

This pipeline uses a simple consensus strategy:

  1. Reverse complement the reverse (R) read so it matches forward (F) orientation
  2. Compare the lengths of F and R reads
  3. Use the longer read as the consensus (typically has better coverage)
  4. In cases where quality differs significantly, the higher quality read is selected

What about single reads?

Samples with only forward or reverse reads are still usable for:

  • Phylogenetic analysis (included in tree construction)
  • Species identification via BLAST
  • Sequence alignment

However, consensus sequences from F+R pairs are generally more reliable.

What if I have unpaired sequences?

Unpaired sequences lack the standard _F or _R naming convention. They are included in downstream analysis but should be reviewed to ensure proper sample naming.
